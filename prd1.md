
每日学习简报：
- 设计思路，参考了zettelkasten和llm wiki。
	- zettlekasten，让知识自己产生新知识，链接即知识，通过链接自动聚集成领域(书由薄变厚)
	- llm wiki，将资料整理成可查询可复用的wiki，快速得到结构(书由厚变薄)，链接用于顺着找文件
	- 我想要的，以前的感悟过一段时间就没了，没有留存在当前的状态，通过周期性回顾用来找到与以前的联系(跨界的抽象联系，方法论这一块也是可以聚集的一个领域)，提高知识的连接性和利用率
- 理想的情况：
	- 一天内，阅读了多篇文章，每一个都添加到若干个source，每一次都尝试能否ingest成一个原子笔记，如果是没有来源的空想，那么放到一种文件夹，如果是有来源的，比如一篇文章的https链接(有联网功能则获取全文)，或者是一个视频的提取的字幕全文文本/llm小结/自己写的一小段话，那么放到另一种文件夹，当天晚上进行回顾(daily子命令)，将成型的想法转为原子笔记(每个原子笔记至少一个链接)，有些source中可能也没有能生成原子笔记的，以及作为skill，可以让llm给出一些高价值的候选项，具体的原子笔记内容还是得自己手敲
	- 第二天回顾的时候仍然会把前一天的未转化成原子笔记的source内容纳入考虑，检查是否能产出原子笔记
	- weekly子命令，相较daily更关注链接的构建
	- monthly，将本月的未产出原子笔记的source再回顾，以及再检查是否有更多的链接
- 形式：CLI+skill，能够方便的集成到claude code/codex，然后也可以在obsidian中查看和编辑
- 当日笔记，分成2026/{202607}/{123}.md，每篇文章也许会带自己的一两句话，以及一些适配obsidian最前面的一些front matter，
- skill，是CLI与llm的缝合处理，先安装skill，然后对应skill配置CLI。skill别写一大段内容，要精简有效
- 子命令汇总，
	- setup，安装CLI，并设置工作目录配置，默认是当前目录，也可以对话设定
	- add，链接(可选)和自己一两句话，如果有联网能能力，那么会获取全文并写一份小结，存入inbox文件夹（闪念笔记），https啥的可以自动提取相关链接；此外还有可能是视频的字幕文本和一段小结，这时候运行add是进行格式的统一
	- ingest，生成笔记（永久笔记），具备原子性，至少有一个链接
	- 回顾(review)，回顾某一段时间内的闪念笔记、文献笔记、永久笔记，
	- daily，列出今天有哪些，然后对每一个遍历进行ingest，会问应该与哪些进行链接并要求你写一段链接原因或其他说明，当然在让我回答前也会一并给出有哪些可能的链接选项
	- weekly/monthly，与daily相似，回顾一遍(调用review)，看是否有新的链接要生成，以及是否有新的笔记要ingest
- CLI要包含的部分，
	- 默认当前工作目录，工作目录的设置，默认当前工作目录，以及可以设置存放目录(默认从当前工作目录开始)，单独给一个slash命令(参考mattpocock/skills的实现)
	- inbox/literature/zettelkasten都要按照年月目录如202607这样组织
	- setup/add/ingest/review
	- daily/weekly/monthly是上面几个动作的语义封装和组合


```text
📁 202607/
├── 📂 00-Inbox/        # 闪念笔记（临时捕捉区）
├── 📂 10-Literature/   # 文献笔记（读书/文章摘要）
├── 📂 20-Zettelkasten/ # 原子笔记（核心知识单元）
```
